The method

Run the ladder. Stop at the first broken rung.

The ladder is a reflex, not a research project. Two rungs work, take the higher one and move on. Fix the first broken rung. Re-run. Continue until the page passes every rung.

Same energy as Ponytail's ladder for code. Applied to design.

  1. System

    Tokens defined before components. Color, type, spacing, elevation, motion. No token, no property.

  2. Type

    Two fonts minimum. Display paired with body. Size contrast of 1.25x or more between steps.

  3. Color

    No lavender-purple. No blue-to-purple gradients. No pure black. Tint everything toward a brand hue.

  4. Layout

    Left-align body text. One strong primitive repeated. No card soup. No colored left borders.

  5. Motion

    Ease-out, not bounce. Transform and opacity only. One orchestrated entrance. Stillness elsewhere.

  6. Content

    Specific headlines. Real metrics. Real photos. No label plus sublabel plus helper text all saying the same thing.

  7. Detection

    Run the script. Score the page. Ship if clean, revise if mild, redesign if heavy or maximum.

Run it

Detection ships as two scripts. Both run.

URL mode loads the page in Playwright and walks the DOM. File mode parses HTML and CSS directly, no browser. Both output the same JSON. Both use the same weighted checks.

# URL mode (Playwright)
$ node cli.mjs detect https://your-site.com
 
# Static file mode (no browser)
$ node cli.mjs detect-files ./src
 
# Output: JSON to stdout, summary to stderr
