# Design Slop Patterns

Comprehensive catalog of AI-generated design patterns with detection criteria. These patterns are not individually bad. They become slop when used by default, without deliberate choice, creating the "generated by a chat interface without an opinion" look.

70+ patterns. Six categories plus a Model-Specific Fingerprints section. Includes the post-Q1 2026 "bone palette" and "italic word-accents" patterns named verbatim in Anthropic's Opus 4.8 prompt guide.

## Typography

### Overused Fonts

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Inter used for everything | The Helvetica of the LLM era. Every generated page defaults to it. | Check `font-family` computed style. Match against `Inter`, `Inter var`. | Pick a distinctive typeface. Geist, Untitled Sans, Söhne, Inktrap, Haas Grotesk for sans-serif. Tiempos, GT Sectra, Freight Text for serif. |
| Space Grotesk for accent words | The AI go-to for "one hero word in a different style." | Check if Space Grotesk is used on less than 5% of text. | Use a genuinely different typeface for emphasis, or use weight and size contrast instead. |
| Roboto, Arial, or system font stack as default | Lazy. Not a choice. | Check for `system-ui`, `-apple-system`, `Arial`, `Roboto` in computed styles. | Make a deliberate font selection. |
| Plus Jakarta Sans, Fraunces, Instrument Serif | Next wave of convergence fonts. Becoming as recognizable as Inter. | Check computed styles against these names. | Deliberate selection outside the overused set. |
| Fraunces / Instrument Serif / Playfair as hero display | The 2026 italic-serif hero reflex. Anthropic's Opus 4.7/4.8 house style. Impeccable: "Anthropic/Google's 28-34% italic-serif-display slop on editorial/food briefs." When you see Fraunces + Inter + cream background, you are looking at Opus 4.7/4.8 output. | Hero `font-family` matches `Fraunces`, `Instrument Serif`, `Playfair Display`, or `Crimson Pro`. | If you want serif display, pick a less-loaded face (Tiempos, GT Sectra, Canela, Newsreader). Or commit to a sans-only hero. |
| Italic word-accents in headings (random italic hero word) | One serif italic word appears inside an otherwise sans-serif hero headline with no semantic reason. It signals "premium editorial" without meaning. The Opus 4.8 prompt guide names this verbatim as "italic word-accents." When *every* sentence has an italic accent, none of them do. | In `h1` or `h2`, find an `em`, `i`, or italic-styled span containing one to three words while the rest of the heading is upright. Flag if the italic word is an abstract noun (trust, clarity, alignment, momentum, craft, signal, intelligence, velocity, resilience, future, commerce, workflows, confidence, idea, everything, power, seamless, effortless, elegant). | Remove the italic treatment unless the word changes tone or meaning. If emphasis is needed, earn it with copy or layout, not typographic seasoning. |
| Decorative bold or italic body emphasis | Body copy bolds or italicizes phrases that do not need emphasis. The formatting does the writing's job. Sister pattern to italic hero word. | Find `strong`, `b`, `em`, or `i` inside paragraphs where the emphasized span is an abstract phrase ("actually moves the needle", "long-term growth", "real outcomes"). | Rewrite the sentence so emphasis is unnecessary. Use formatting only for terms, titles, warnings, or genuinely contrastive emphasis. |
| Unicode faux formatting | Mathematical Unicode letters render as 𝗯𝗼𝗹𝗱 or 𝘪𝘵𝘢𝘭𝘪𝘤 instead of semantic HTML. Also catches decorative arrows `→`, multiplication separators `×`, and repeated dot separators `·` in prose. | Detect characters in `U+1D400–U+1D7FF`, plus decorative arrows `→`, multiplication separators `×`, or repeated dot separators `·` in prose. | Use semantic HTML for real emphasis. Remove decorative Unicode in product copy, docs, and professional prose. |
| Tracking-tight oversized hero headline | `letter-spacing: -0.04em` or tighter on a 64-96px hero. The "modern premium" reflex. Impeccable flags this as "Crushed letter spacing." | Computed `letter-spacing` below -0.03em on elements with `font-size` above 56px. | Use -0.01 to -0.02em on large headings only. Or zero tracking and let the typeface breathe. |
| Section eyebrows as italic-serif kicker labels | "✦ *Introducing* / [Product name]" — italic-serif tracked-letterspaced labels above section headings. Anthropic's own `impeccable.style` flags this exact pattern as "Hero eyebrow / pill chip." | Small text element with `font-style: italic`, `text-transform: uppercase` OR `letter-spacing` above 0.08em, in a serif font, immediately preceding a heading. | Cut the eyebrow. Or replace with a numbered section marker (01 / 02 / 03) only if the section is genuinely sequential. |
| Numbered section markers (01 / 02 / 03) on non-sequential sections | AI editorial-scaffold reflex. Earns its place only when the section actually is a sequence. | Two-digit numeric prefix above section heading on non-sequential sections (about, pricing, testimonials, contact). | Remove the numbers. They imply sequence the page doesn't have. |

### Hierarchy Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Single font family for entire page | No typographic hierarchy. Headings, body, labels, buttons all look the same. | Count unique `font-family` values. If 1, flag. | Pair a distinctive display font with a refined body font. Two families minimum. |
| Font sizes too close together | Flat hierarchy. No visual priority. | Calculate ratio between consecutive font sizes. If max ratio below 1.2, flag. | Use 1.25x ratio between size steps. Fewer sizes with more contrast. |
| Monospace as "technical" shorthand | Lazy stereotype to signal "developer" vibes. | Check if monospace is used on non-code elements. | Pick real type choices that communicate the right tone. |
| All-caps body text | Hard to read in long form. Destroys word shape recognition. | Check for `text-transform: uppercase` on elements with more than 5 words. | Reserve uppercase for short labels of 2 to 3 words. |
| Tight line height below 1.3x | Multi-line text crammed. Hard to track lines. | Check `line-height` divided by `font-size` ratio. Flag if below 1.3 on multi-line elements. | Use 1.5 to 1.7 for body text. |

## Color

### AI Default Palette

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Lavender-purple accents | The most recognizable AI color tell of 2023-2025. Still the second-most common after bone. Adam Wathan's `bg-indigo-500` Tailwind UI default (apologized for Aug 2025) propagated through training data. | Check for colors in HSL range H: 250 to 290, S: 50 to 100%, L: 50 to 80%. | Choose earth tones, cream-and-pink, high-contrast black-and-bright, warm reds, cool greens. |
| Blue-to-purple gradients | Default "modern" gradient. | Check for `linear-gradient` or `radial-gradient` transitioning from blue range to purple range. | Use solid colors or intentional gradients tied to brand palette. |
| Bone / cream / off-white background as default | The post-2025 Opus 4.7/4.8 house style. The 2026 replacement for VibeCode Purple. Anthropic's prompt guide names it verbatim: "warm cream/off-white backgrounds (~#F4F1EA)." Identical across vet clinic, fintech, and crypto. When 22% of Show HN pages trigger 4+ AI design tells (Krebs audit), this palette is the single biggest contributor. | Background hex within ±3 points of `#F4F1EA`, `#F4F3EE`, `#F6F5F2`, or `#FAF8F2` AND body text in `#191817`-`#37352F` warm-charcoal range AND accent in terracotta (`#C96442` / `#C8643C`), sage (`#8A9A7B`), dust rose (`#C9A8A0`), or amber (`#C98A42`). | Commit to a real brand color. If you want warm cream, pick a different accent (indigo `#5266EB`, orange `#FF6719`, deep teal `#0D7377`, oxblood `#7B1E1E`) so the cream doesn't read as default Opus. Or name the aesthetic family explicitly. |
| Warm-charcoal text on cream failing WCAG AA | The bone palette's accessibility failure mode. `#5A5548` "warm gray" text on `#F4F1EA` cream is contrast 5.0:1 — passes AA for body but fails AAA, fails AA for small text below 18px. | Compute contrast ratio. Flag if body text on bone background uses `#5A5548` or lighter for text below 18px. | Use `#191817` or `#2A2924` for all body text on cream. Reserve `#5A5548` for muted secondary only. |
| "Bone + dark brown" dark mode | Dark mode using `#2A2520` text on `#1A1714` background — warm browns that fail contrast. The bone palette leaking into dark mode with worse contrast. "Dark-mode sites where the text and subtext are various shades of dark brown or beige. It looks awful. It breaks accessibility standards." (u/LeoRiley6677, r/ClaudeCode) | Dark mode (background luminance below 0.2) with text color in the warm-brown range (`#1A1000`-`#3D2E20`) or beige text (`#E8D8C0`-`#F0E8D8`). | Use a tinted near-black background (`oklch(15% 0.02 250)`) and pure off-white text (`#FAFAFA`). Don't ship warm-brown-on-warm-brown dark mode. |
| Permanent dark mode with medium-grey body text | Background `#0B0B0F`, body text `#9CA3AF`, all-caps section labels in `#A0A0A0`. Krebs audit: 34% of Show HN pages — the single most common tell. | Background luminance below 0.2 AND body text color in `#808080`-`#A0A0A0` range (medium grey). | Use `#D4D4D8` or lighter for body text on dark backgrounds (contrast 9:1+). Or commit to a light theme. |
| Gradient text | Decorative. Reduces scannability. Common on headings and metrics. | Check for `background-clip: text` plus gradient background. | Use solid colors for all text. |
| Neon box-shadow glows on dark backgrounds | Cyberpunk-by-default. | Check for `box-shadow` with spread above 10px and high saturation color on dark elements. | Use subtle, purposeful lighting. Remove glows. |
| shadcn/ui default gray palette | Every AI-generated site ships with the same gray scale. | Check for exact shadcn gray values (`hsl(0 0% 3.9%)` through `hsl(0 0% 98%)`). | Customize color tokens. Tint grays toward brand hue. |
| Pure black (#000000) background | Harsh, unnatural. | Check for exact `#000000` or `rgb(0,0,0)` on backgrounds. | Tint toward brand hue (e.g. oklch(12% 0.01 250)). |
| Gray text on colored backgrounds | Washed out, hard to read. | Check text color on colored backgrounds. Compute contrast. Flag if below 4.5:1. | Use darker shade of background color, or white and near-white. |
| Tailwind-default indigo + slate + gray | `bg-indigo-500`, `text-slate-900`, `bg-gray-50`. Tailwind defaults leaked into training data. The single most common AI landing-page palette of 2024. | Check CSS for Tailwind class names matching `indigo-500`, `slate-900`, `gray-50`, or hex equivalents. | Define your own palette. Pick a different accent. Tint the grays. |

### Background Patterns

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Aurora or mesh gradient backgrounds | The 2024 default hero background. Three blurred color blobs floating behind text. | Check for multiple `radial-gradient` declarations or large blurred elements with high-saturation colors in the background. | Use a solid tinted background. If you need depth, use a single subtle gradient tied to brand palette. |
| Animated gradient background | Gradient that shifts hues over time. Distracting. | Check for `@keyframes` animating `background-position` or `--gradient-angle` custom property. | Static background. If motion is needed, animate something else. |

## Layout

### Template Patterns

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Everything centered | Symmetrical, safe, no opinion. | Check `text-align` across major elements. Flag if more than 80% are `center`. | Left-align body text. Use asymmetric layouts. Center only hero and CTAs. |
| Badge above hero H1 | AI template default. | Check for small pill or badge element immediately preceding the H1 in DOM order. | Remove badge. Or make it serve a genuine purpose with distinctive styling. |
| Copy-paste section templates | Hero then metrics then features then testimonials then pricing then CTA. | Count distinct section layout patterns. Flag if below 3 patterns across 6 or more sections. | Vary section structures. Mix narrative with feature grids. Break the conveyor belt. |
| Hero metric layout | Big number plus small label repeated 3 or 4 times. | Check for elements with very large font-size adjacent to very small text, repeated in a row. | Only show real, meaningful, verifiable metrics. Give them room or remove them. |
| Monotonous spacing | Same gap everywhere. No rhythm. | Compute standard deviation of spacing values. Flag if below 2 unique values across sections. | Tight grouping for related items (8 to 12px). Generous gaps between sections (48 to 96px). |
| Centered hero trio | Badge plus H1 plus subhead plus two CTAs, all centered. The universal AI homepage. | Check hero section for badge followed by h1 followed by paragraph followed by two buttons, all text-align center. | Break the template. Left-align the hero. Cut the badge. One CTA. |
| Centered hero with two CTAs (primary + "Talk to sales") | Primary CTA "Start free trial" + secondary ghost button "Talk to sales". Default AI hero button pair. Universal across v0/Lovable/Bolt output. | Centered hero section with 2+ buttons where one matches `/^(Get Started|Try Free|Start Building|Start Free|Sign Up)$/i` and another matches `/^(Talk to Sales|Book a Demo|Contact Sales|Learn More)$/i`. | One CTA. Or two CTAs with specific verbs tied to the product ("Run your first deploy" / "Read the docs"). |
| Mac mockup or isometric 3D illustration as hero image | Floating laptop/screenshot with slight 3D tilt + soft shadow. Default v0 / Lovable / Bolt hero visual. | Hero section contains an `<img>` with class matching `/mockup|screenshot|hero-image|product-shot/` AND `transform: rotate(...)` or `perspective` applied. | Use a real product screenshot, no rotation. Or omit the hero image entirely and let the headline carry. |
| Marquee logo carousel | Infinite-scrolling row of grayscale client logos. Animated, distracting, never informative. | CSS `@keyframes` animating `transform: translateX(...)` on a row of `<img>` elements with `filter: grayscale(...)`. | Static logo grid. Or remove entirely if logos don't link to case studies. |
| Animated count-up stats | "10M+ users" with JS counter animation from 0. Distracting, slows page, no informational value. | JS or CSS animation that animates a numeric value from 0 to a target over time. | Display the final number. Skip the animation. |
| Generic abstract geometric / aurora blobs as hero illustration | Blurred colored blobs as hero illustration. Same on every site. Default LLM hero visual. | Hero `<div>` or `<svg>` with `filter: blur(...)` and high-saturation fill colors, no real content. | Real product screenshot. Real photography. Or strong typography alone. |
| "Trusted by" grayscale logo strip | Row of desaturated company logos below the hero. Almost never linked to case studies. | Check for a row of `img` elements with `filter: grayscale(100%)` or `opacity: 0.5` immediately after the hero. | Cut the strip unless every logo links to a verifiable case study. Otherwise, remove. |
| Inconsistent radii across the page | `rounded-xl` here, `rounded-2xl` there, `rounded-full` elsewhere. The model doesn't have a radius system. Real design systems commit to one or two radii. | Count unique `border-radius` values across cards, buttons, inputs. Flag if more than 3 distinct values without a documented scale. | Commit to 2-3 radii max. E.g. `--radius-sm: 4px`, `--radius-md: 8px`, `--radius-pill: 9999px`. |
| All-caps eyebrow with dots or slashes | `NEW · AI OPS · PRIVATE BETA` or `SYSTEM // DESIGN // OPS` appears above the hero. It looks designed but usually carries no user value. | Small all-caps text before `h1` containing `·`, `•`, `//`, or 3+ short labels. | Keep only if it is a real category, status, or metadata label. Otherwise cut it. |
| Premium-startup poster composition | Oversized type, tiny metadata blocks, editorial italics, badges, and abstract product mockups combine into a poster that says little. | Cluster check: hero has uppercase eyebrow, italic heading span, two CTAs, mock dashboard image, and abstract nouns. | Start from the product claim. Name the user, action, input, output, and constraint. Rebuild the composition around that. |
| Bento grid of vague benefits | Four to six polished cards describe `Adaptive Intelligence`, `Seamless Integration`, `Actionable Insights`, or similar empty benefits. | Section has 4+ card-like siblings with short headings and descriptions containing vague benefit terms. | Replace benefits with concrete jobs, screenshots, examples, limits, or proof. Vary card structure only when content demands it. |

### Card Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Colored left/top border on cards, blockquotes, or code blocks | Most reliable AI design tell. Also leaks onto blockquotes (`blockquote { border-left: 4px solid #10B981; }`) and code blocks (`pre { border-left: 3px solid var(--accent); }`). The colored vertical stripe is the universal LLM "I tried to make it look designed" reflex. | Check for `border-left` or `border-top` with colored value on elements with `border-radius` (cards) OR on `blockquote` / `pre` / `code` elements. Flag if thick (>2px) and colored. | Remove border. Use subtler accents (background tint, italic, quote mark) or none. For code blocks: use background color alone. For blockquotes: use indentation + italic, no border. |
| Identical feature cards (icon plus heading plus text) times N | Default AI homepage. | Count sibling elements with identical dimensions, same child structure, same padding. Flag if 3 or more identical siblings. | Vary card sizes and content density. Mix full-width and half-width. |
| Cards inside cards (Cardocalypse) | Visual noise from nesting. | Check DOM depth of elements with border, shadow, or background. Flag if cards nested 3 or more levels deep. | Flatten hierarchy. Use spacing, typography, dividers. |
| Wrapping everything in cards | Every content block gets a bordered container. | Count bordered elements versus total content blocks. Flag if more than 70% are card-wrapped. | Use spacing and alignment for grouping. Cards only for interactive or elevated content. |
| Massive icon containers | Decoration bigger than message. | Compare icon container dimensions to adjacent text content. Flag if icon area larger than text area. | Side-by-side icon and heading. Or icon in flow without container. |
| Icon tile stacked above heading | Universal AI feature-card template. | Check for rounded-square container directly above a heading element in DOM. | Vary the pattern. Side-by-side layout, or no container. |
| Border accent on rounded element | Thick accent border clashes with rounded corners. | Check for `border` above 2px colored on elements with `border-radius` above 8px. | Remove border or remove radius. They clash visually. |
| v0 or Lovable component DNA | Components lifted from AI generators carry their default styling: rounded-xl, soft shadow, Inter font, gray-50 background. | Check for the combination of `rounded-xl`, `shadow-sm`, `bg-gray-50`, and Inter font on the same component. | Override every default. Restyle to match your design system. |

### Sequence Patterns

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Numbered "1, 2, 3" step sequences | Conveyor-belt layout. | Check for sibling elements containing sequential numbers as content or aria-labels. | Use timeline, narrative, comparison, accordion. |
| Stat banner rows | Row of unverifiable metrics. | Check for sibling elements with large number plus small label pattern in a horizontal row. | Only include real, meaningful metrics. Give them proper space or remove. |
| Sidebar or nav with emoji icons | Emoji as icon substitute. No design system. | Check for emoji characters in navigation or sidebar text content. | Use real icon library (Phosphor, Tabler, Lucide) or skip icons. |
| All-caps section headings | Shouting. Hard to read. | Check for `text-transform: uppercase` on heading elements. | Reserve for short labels of 2 to 3 words. Title or sentence case for headings. |
| 3-tier pricing with "Most Popular" badge | Default AI pricing page. The badge carries no information. | Check for three pricing cards with one marked "Most Popular" or "Recommended." | Cut the badge unless it encodes real data. Differentiate tiers by real features. |
| FAQ accordion before footer | Default AI page ending. Nobody reads it. | Check for an accordion component immediately before the footer containing question elements. | Move FAQs to a real help page. Or answer the top question inline in the body. |

### Structural Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Line length too long (above 80ch) | Eye loses place tracking back. | Measure line length on text blocks. Flag if above 80ch average. | Add max-width 65ch to 75ch to text containers. |
| Skipped heading levels | Breaks document outline for screen readers. | Check heading hierarchy. Flag if h1 to h3 without h2. | Use proper heading hierarchy. No skipping. |
| Cramped padding | Text too close to container edge. | Check padding on bordered or colored containers. Flag if below 8px. | Minimum 8px padding (12 to 16px ideal) inside bordered containers. |
| Tiny body text (below 12px) | Hard to read, especially on high-DPI screens. Often paired with low-contrast muted color (`#737373` / `text-muted`) making the problem worse. | Check `font-size` on body text. Flag if < 12px. Flag if < 14px on text colored `#737373` or lighter. | Minimum 14px for body content. 16px ideal. Never ship 11-12px gray meta text — it's the AI's "I forgot to write real copy so here's a caption" reflex. |
| Amputated features on mobile | Critical functionality hidden on mobile. | Check for `display: none` or `visibility: hidden` on key interactive elements at mobile breakpoints. | Adapt the interface to context. Do not strip features. |
| Modal abuse | Complex content in a modal. | Check modal elements. Flag if scrollable, multi-column, or containing more than 5 interactive elements. | Complex content deserves its own page. |

### Structural Problems (continued)

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Useless sticky header with in-page anchor buttons only | Sticky top nav with 4-5 buttons that all scroll to sections on the same page (`#detector`, `#tells`, `#models`). No real navigation. No way to leave the page. Pure scroll-jumping theater. | Sticky/fixed header (`position: sticky` or `fixed`) containing only anchor links (`href^="#"`) and no real outbound links. | Cut the sticky header. Or replace with real navigation (link to docs, blog, signup, source). If the page genuinely needs in-page anchors, put them in a sidebar or table of contents, not a fake nav bar. |
| Footer meta-commentary (version strings, "this demo deliberately avoids...") | Footer text that comments on the design itself instead of carrying real entity info. Includes "This demo deliberately avoids the bone palette. Background #FFFFFF..." style captions, version strings like "slop-buster v1.7.0 · MIT · 2026", and "Built with X" attributions. The footer should carry real entity name, real license, real contact info — not narrate its own design choices. | Footer text matching `/deliberately avoids/i` OR `/this demo (deliberately|avoids)/i` OR `/background #/i` OR `/accent #/i` OR version strings matching `/v\d+\.\d+\.\d+.*·.*MIT/i` in footer. | Real entity name. Real city. Real license. Or omit. Cut any footer text that comments on the design itself. |


## Motion and Interaction

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Bounce or elastic easing | Dated and tacky. | Check for `cubic-bezier` values matching bounce or elastic patterns, or `animation-timing-function: bounce`. | Use ease-out-quart `cubic-bezier(0.25, 1, 0.5, 1)` or ease-out-expo `cubic-bezier(0.16, 1, 0.3, 1)`. |
| Glassmorphism everywhere | LLM default since 2022. Decorative, not functional. | Check for `backdrop-filter: blur()` combined with semi-transparent backgrounds on multiple elements. | Use only where it solves a real layering problem. Otherwise, solid backgrounds. |
| Scroll-jacking | The page hijacks scroll position. Disorienting. | Check for `scroll-snap-type` with mandatory snap points, or JS intercepting wheel events. | Let the user control scroll. Remove scroll-snap or set to `proximity`. |
| Custom cursor | Replaces the system cursor. Hostile on touch devices. | Check for `cursor: none` combined with a custom element following mouse position. | Remove. The system cursor is the right cursor. |
| AI sparkle badge | The sparkles emoji used as a "this is AI" indicator. | Check for `✨` character in badges or button labels. | Cut the emoji. State what the AI does in words. |
| Rocket emoji in SaaS hero | The "🚀" emoji in "Launch your product" CTAs. | Check for rocket emoji in hero or CTA text. | Cut the emoji. Use a real verb. |
| Every button is primary | No hierarchy. Everything shouts. | Count button elements. Check if more than 50% share the same visual weight. | One primary action per view. Use ghost, text-link, secondary styles for hierarchy. |
| Animating width, height, padding, or margin | Layout thrash and janky performance. | Check CSS transitions and animations for layout properties. | Use transform and opacity. Or `grid-template-rows` for height. |
| Infinite scroll | Content never ends. No footer. No way to stop. | Check for scroll-triggered content loading with no end sentinel. | Pagination or "load more" button. Give the user a way to stop. |

## Content and Copy

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Vague aspirational headline | Could describe any company. Communicates nothing. "Powering the future of [industry]." | Can this headline appear on 100 or more different websites? If yes, slop. | State what the product does, specifically. Name the problem solved. |
| Generic verb-first hero copy ("Build / Ship / Scale / Transform / Unlock") | The LLM's reflex when given no brief. Could describe any company. The model's reflex when given no brief is to start with a verb. | Hero H1 starts with one of: Build, Ship, Scale, Transform, Unlock, Supercharge, Empower, Streamline, Harness, Leverage, Reimagine, Elevate, Accelerate, Amplify. | State what the product does, specifically. E.g. "Your deploy pipeline runs in 12 seconds." |
| Aphoristic-cadence headline ("Not X. *Y.*") | Two-clause headline landing on a short italic-serif rebuttal. Anthropic's `impeccable.style` flags this as "Aphoristic-cadence copy." Examples: "It's not a feature. *A platform.*" "Days fade. *Dots don't.*" | Regex on hero/subhead text for pattern `^[A-Z][^.]{3,30}\.\s+<[em]i?>[A-Z][^.]{1,30}\.</[em]i?>` | Say what the product does. Don't perform wit. |
| Footer "Made with ♥ in [city]" or "© 2026 [Generic]" | Default LLM footer. Universal across generated sites. | Footer text matching `/Made with [♥❤] in [A-Z][a-z]+/i` OR `/© \d{4} [A-Z][a-z]+/i` with no real entity name. | Real entity name. Real city. Real license. Or omit. |
| "Get Started" / "Try Free" / "Start Building" CTAs with arrow icon | The model's default CTA vocabulary. Could be any product. | Button text matching `/^(Get Started|Try Free|Start Building|Start Free|Try .* Free|Get .* Free)$/i` AND containing an arrow icon (`→`, `→`, `➜`, `↗`). | Use a verb specific to the product. "Run your first deploy" beats "Get Started." |
| Pricing copy: "/month billed annually" + "Save 20%" | The model's reflex for pricing UX. Universal across AI-generated pricing pages. | Pricing card text matching `/\/month billed annually/i` OR `/Save \d+%/i`. | State the actual price. State the actual billing period. Skip the marketing overlay. |
| "Powering the future of [industry]" hero headline | Generic aspirational headline with no specifics. "AI generates them by averaging every headline it has seen." (925studios) | Hero text matching `/Powering the future of/i` OR `/Reimagine \w+/i` OR `/The future of/i`. | Replace with a specific claim about what the product does. |
| Redundant UX writing | Label plus sublabel plus helper text all saying the same thing. | Check sibling elements inside a container. If text content overlap above 70%, flag. | Say it once, say it well. Pick the clearest phrasing. |
| "Trusted by 10,000+ companies" with no logos | Unverifiable claim. No logos that link. | Check for trust claims with no corresponding linked logo strip. | Cut the claim. Or link to a real customer list with case studies. |
| Fake 5-star testimonials | Star rating with no real reviews behind it. | Check for star rating elements with no link to a review platform. | Remove. Or link to the actual review source (G2, Capterra, App Store). |
| Generic feature descriptions | "Quality Guarantee" and "Secure Payment" with no specifics. | Check feature text for generic claims with no specifics. | State the specific guarantee. State the specific payment methods. |
| "Most Popular" tier with no reason | The middle pricing tier is "Most Popular" with no data to back it. | Check pricing for "Most Popular" badge with no usage data. | Cut the badge. Or back it with real conversion data. |
| Generic about page | "We are a team of passionate innovators..." with no names, no photos. | Check about page for vague team descriptions with no real people. | Real names. Real photos. Real roles. Or cut the page. |
| Random testimonial avatars from randomuser.me / pravatar.cc | Stock-photo avatars on testimonials. ## Model-Specific Fingerprints (Post-Q1 2026)

When you identify slop, name the model that likely produced it. Different models have different defaults and need different overrides. The fingerprints below are drawn from Anthropic's Opus 4.8 prompt guide, Caylent's Opus 4.7 deep-dive, the Muzli one-week-in review, the GLM-4.x release notes, and the DeepSeek V4 changelog.

### 8.1 Anthropic Claude Opus 4.5 (mid/late 2025) — "VibeCode Purple" era

**Defaults:** Inter everywhere, purple gradients on white, centered hero with eyebrow + headline + subhead + two CTAs, 3-up icon feature cards, shadcn-default styling.

**Tell density:** Adrian Krebs' Show HN audit (1,590 pages, April 2026) — which largely reflects 4.5-era output — found 22% of pages heavy slop, 32% mild, 46% clean. Most common single tell: permanent dark theme (34%), gradient backgrounds (27%), icon-card grids (22%).

**Italic / serif tendency:** Moderate. The serif-italic hero word emerges here as the "I tried" upgrade from pure Inter.

**Override:** Explicit light-mode brief, named font pairing, specific accent color (not indigo, not purple). Provide a DESIGN.md with `--bg: #FFFFFF` and `--text: #1A1A1A`.

### 8.2 Anthropic Claude Opus 4.6 (early 2026)

**Defaults:** Similar to 4.5 but with stronger tool use and longer context. The bone-cream palette begins to emerge as a house tendency but is not yet dominant.

**Italic / serif tendency:** Stronger than 4.5. The Fraunces/Instrument Serif hero word is now reflexive.

**Override:** Same as 4.5 plus explicit anti-bone-palette constraint. "Don't use cream" is not enough — specify a concrete alternative palette.

### 8.3 Anthropic Claude Opus 4.7 (released April 17, 2026 alongside Claude Design)

**Defaults:** The "warm editorial" house style arrives fully formed. Caylent: "Opus 4.7 appears to have a persistent default design taste: warm cream or off-white backgrounds, serif display typography, italic accents, and terracotta or amber highlights."

**Caylent again:** "If you use Opus 4.7 to generate frontend code, slide decks, or visual variants, generic instructions like 'clean and minimal' may not be enough to break the default style."

**Muzli one-week-in review:** "Opus 4.7 has its own house style it falls back to (warm cream backgrounds, Fraunces-style serifs, terracotta accents), beautiful for hospitality and editorial, wrong for fintech and dashboards. Override it explicitly."

**Italic / serif tendency:** Very strong. This is the model that makes "random words in italics" a fingerprint.

**Behavioral shift:** "The model also uses tools less often by default, spawns fewer subagents, and has an opinionated design aesthetic — warm cream, serif type..." (natesnewsletter, Apr 21 2026).

**Override:** Provide a DESIGN.md with explicit palette + typography tokens. Use Anthropic's `<frontend_aesthetics>` anti-slop prompt. Have the model propose 3 options before building. Specify concrete alternatives ("Use pale silver-gray tones deepening into blue-gray, with 4px corner radius and a square angular sans-serif"), not negatives ("Don't use cream").

### 8.4 Anthropic Claude Opus 4.8 (post-Q1 2026, current)

**Defaults:** Per the official Opus 4.8 prompt guide: "Claude Opus 4.8 has strong design instincts, with a consistent default house style: warm cream/off-white backgrounds (~#F4F1EA), serif display type (Georgia, Fraunces, Playfair), italic word-accents, and a terracotta/amber accent. This reads well for editorial, hospitality, and portfolio briefs, but will feel off for dashboards, dev tools, fintech, healthcare, or enterprise apps. The default appears in slide decks as well as web UIs."

**Persistence:** "This default is persistent. Generic instructions ('don't use cream,' 'make it clean and minimal') tend to shift the model to a different fixed palette rather than producing variety."

**Anthropic's recommended fix:** "Have the model propose options before building. This breaks the default and gives users control."

**Anti-slop fragment Anthropic ships:** "NEVER use generic AI-generated aesthetics like overused font families (Inter, Roboto, Arial, system fonts), cliched color schemes (particularly purple gradients on white or dark backgrounds), predictable layouts and component patterns, and cookie-cutter design that lacks context-specific character."

**Italic / serif tendency:** Maximum. The italic word-accent is now part of the named house style.

**Override:** Same as 4.7 plus: have the model propose 3 options before building. Specify concrete alternatives, not negatives. Drop the anti-slop fragment into the project's CLAUDE.md or DESIGN.md verbatim.

### 8.5 GLM 4.5 / 4.6 / 4.7 / 5.x

**GLM-4.5 (z.ai/blog/glm-4.5):** "The frontend interfaces generated by GLM-4.5 exhibit enhanced functionality and aesthetic appeal, demonstrating strong alignment with human preferences."

**GLM-4.6 (zai-org/GLM-4.5 README):** "improvements in generating visually polished front-end pages."

**GLM-4.7 (README):** "GLM-4.7 takes a big step forward in improving UI quality. It produces cleaner, more modern webpages and generates better-looking slides with more accurate layout and sizing."

**GLM-5 (arxiv.org/html/2602.15763v1):** "GLM-5, a next-generation foundation model designed to transition the paradigm of vibe coding to agentic engineering."

**Observed fingerprints:** Less specifically documented than Claude, but community reports (r/LocalLLaMA, Medium review by leucopsis) note "more polished, human-like outputs in front-end development and UI design." GLM tends toward a slightly different palette — Chinese-influenced minimalism, more use of Toss-blue / cool grays, less italic-serif hero. The "cleaner, more modern" phrasing in release notes suggests GLM is reaching for the same Stripe/Linear editorial-minimalism center as Claude, but with a cooler color temperature.

**Italic / serif tendency:** Lower than Opus 4.7/4.8. GLM tends toward sans-only heroes.

**Override:** Specify cool palette (Toss-blue, sage, cool gray). Name a Chinese-minimalism or Swiss-editorial reference. Provide specific hex codes.

### 8.6 DeepSeek V4 Pro (released April 24, 2026)

**DeepSeek API docs:** "Generated web pages and games now feature improved aesthetics. Reduced Hallucinations. Significantly suppressed hallucination issues present in legacy R1..."

**DeepSeek V4 Pro:** 1.6-trillion parameter MoE model, MIT license.

**DeepSeek V4 design language:** Less documented than Claude/GLM. Community reports suggest DeepSeek V4 leans toward functional dense dashboards (data-heavy aesthetic) rather than editorial cream/serif. It tends to use Inter + a saturated single accent (often DeepSeek's brand blue) and avoids the italic-serif hero.

**Italic / serif tendency:** Lowest of the three frontier model families for design generation.

**Override:** Usually fine. Watch for overuse of DeepSeek brand blue on non-DeepSeek sites.

### 8.7 v0 / Lovable / Bolt / Google Stitch / Claude Design

**v0 by Vercel:** "v0's pictures all blend in, lacking unique characteristics or originality. every v0 generation looks like the same bland, emotionless page." (community.vercel.com) Default: shadcn/ui + Inter + slate gray + indigo accents. Most convergent of all tools.

**Lovable:** Strongest "purple gradient + Inter + 3-card grid" fingerprint of the consumer tools. Default output is the "Purple Problem" exemplar. Mac mockup + aurora blob + Inter + indigo CTA is the most-cited Lovable output fingerprint.

**Bolt:** Similar to Lovable but with more variety when explicitly prompted. Bolt's 2026 blog: "Learn how to build a website that stands out (and doesn't scream 'AI-generated') in this guide to scroll-stopping designs for 2026."

**Google Stitch:** Origin of the DESIGN.md spec; open-sourced the format. Stitch output tends toward Material-3-adjacent defaults — Roboto / Google Sans, Material elevation, Google brand colors.

**Claude Design (claude.ai/design):** Powered by Opus 4.7. Ships with the warm-cream/terracotta/Fraunces house style. The Muzli review and adpharm teardown both document that without a DESIGN.md and explicit anti-slop prompt, Claude Design output converges on this house style across industries. Mac mockup + cream background + Fraunces is the Claude Design fingerprint.

**Override (all tools):** Provide a DESIGN.md with explicit color tokens, typography, radius, and elevation. Use Anthropic's anti-slop fragment. Name an aesthetic family explicitly (Editorial Minimalism, Terminal-Core, Warm Editorial, Data-Dense Pro, Cinematic Dark, Playful Color, Glass / Soft-Futurism, Neon Brutalist, Cult / Indie — per the awesome-claude-design catalog). Replace Mac mockups with real screenshots. Replace aurora blobs with solid backgrounds.
