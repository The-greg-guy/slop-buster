# Design Slop Patterns (Advanced)

Comprehensive catalog of AI-generated design patterns with detailed detection criteria. These patterns are not individually bad — they become slop when used by default, without deliberate choice, creating the "generated by a chat interface without an opinion" look.

## Typography

### Overused Fonts

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Inter used for everything | The Helvetica of the LLM era. Every generated page defaults to it. | Check `font-family` computed style. Match against `Inter`, `Inter var`. | Pick a distinctive typeface. Geist, Untitled Sans, Söhne, Inktrap, Haas Grotesk for sans-serif. Tiempos, GT Sectra, Freight Text for serif. |
| Space Grotesk for accent words | The AI's go-to for "one hero word in a different style." | Check if Space Grotesk is used on <5% of text (accent only). | Use a genuinely different typeface for emphasis, or use weight/size contrast instead. |
| Geist Serif italic for accent words | Same role as Space Grotesk. Another overused accent. | Check for Geist Serif in italic on isolated words. | Same fix. Use weight/size contrast or a genuinely distinct face. |
| Roboto, Arial, or system font stack as default | Lazy. Not a choice. | Check for `system-ui`, `-apple-system`, `Arial`, `Roboto` in computed styles. | Make a deliberate font selection. Even system fonts work if paired intentionally. |
| Plus Jakarta Sans, Fraunces, Instrument Serif | Next wave of convergence fonts. Becoming as recognizable as Inter. | Check computed styles against these names. | Same fix. Deliberate selection outside the overused set. |
| Fraunces / Instrument Serif / Playfair as hero display | The 2026 italic-serif hero reflex. Anthropic's Opus 4.7/4.8 house style. Impeccable: "Anthropic/Google's 28-34% italic-serif-display slop on editorial/food briefs." | Hero `font-family` matches `Fraunces`, `Instrument Serif`, `Playfair Display`, or `Crimson Pro`. | If you want serif display, pick a less-loaded face (Tiempos, GT Sectra, Canela, Newsreader). Or commit to a sans-only hero. |
| Italic word-accents in headings | The Opus 4.8 prompt guide names this verbatim: "italic word-accents." Manifests as `<em>` or italic-serif words mid-sentence ("Build *your* next *idea* with *confidence*"). Verbal fry in typographic form. | `<em>` tags inside `<h1>`, `<h2>`, hero containers, feature card titles, CTA labels. Especially when paired with serif font-family. | Rewrite the headline in roman. If emphasis is needed, use weight contrast (800 vs 400) or size contrast (3x). Cut italic from feature copy entirely. |
| Tracking-tight oversized hero headline | `letter-spacing: -0.04em` or tighter on a 64-96px hero. The "modern premium" reflex. | Computed `letter-spacing` below -0.03em on elements with `font-size` above 56px. | Use -0.01 to -0.02em on large headings only. Or zero tracking and let the typeface breathe. |

### Hierarchy Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Single font family for entire page | No typographic hierarchy. Headings, body, labels, buttons all look the same. | Count unique `font-family` values across all elements. If 1, flag. | Pair a distinctive display font with a refined body font. Minimum two families. |
| Font sizes too close together | Flat hierarchy. No visual priority. | Calculate ratio between consecutive font sizes. If max ratio < 1.2, flag. | Use at least 1.25× ratio between size steps. Fewer sizes with more contrast. |
| Monospace as "technical" shorthand | Lazy stereotype to signal "developer" vibes. | Check if monospace is used on non-code elements (headings, labels). | Pick real type choices that communicate the right tone. |
| All-caps body text | Hard to read in long form. Destroys word shape recognition. | Check for `text-transform: uppercase` on elements with >5 words. | Reserve uppercase for short labels (2–3 words). Use title/sentence case for everything else. |

### Spacing Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Tight line height (below 1.3×) | Multi-line text crammed. Hard to track lines. | Check `line-height` / `font-size` ratio. Flag if < 1.3 on multi-line elements. | Use 1.5–1.7 for body text. |
| Wide letter-spacing on body (above 0.05em) | Disrupts natural character groupings. Slows reading. | Check `letter-spacing` on body text elements. Flag if > 0.05em. | Reserve wide tracking for short uppercase labels only (0.05–0.12em). |
| No letter-spacing variation | All text same tracking. No differentiation. | Count unique `letter-spacing` values. If 1 (or 0), flag. | Tight on large headings (-0.01 to -0.02em). Default on body. Open on labels (0.04–0.08em). |

## Color

### AI Default Palette

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Lavender-purple accents | The most recognizable AI color tell of 2023–2025. | Check for colors in the HSL range H: 250–290, S: 50–100%, L: 50–80%. Flag purple-ish accents. | Choose earth tones, cream-and-pink, high-contrast black-and-bright, warm reds, cool greens. |
| Blue-to-purple gradients | Default "modern" gradient. | Check for `linear-gradient` or `radial-gradient` transitioning from blue range to purple range. | Use solid colors or intentional gradients tied to brand palette. |
| Bone / cream / off-white background as default | The post-2025 Opus 4.7/4.8 house style. The 2026 replacement for VibeCode Purple. Anthropic's prompt guide names it verbatim: "warm cream/off-white backgrounds (~#F4F1EA)." Identical across vet clinic, fintech, and crypto. | Background hex within ±3 points of `#F4F1EA`, `#F4F3EE`, `#F6F5F2`, or `#FAF8F2` AND body text in `#191817`–`#37352F` warm-charcoal range AND accent in terracotta (`#C96442` / `#C8643C`), sage (`#8A9A7B`), dust rose (`#C9A8A0`), or amber (`#C98A42`). | Commit to a real brand color. If you want warm cream, pick a different accent (indigo `#5266EB`, orange `#FF6719`) so the cream doesn't read as default Opus. Or name the aesthetic family explicitly. |
| Warm-charcoal text on cream failing WCAG AA | The bone palette's accessibility failure mode. `#5A5548` "warm gray" text on `#F4F1EA` cream is contrast 5.0:1 — passes AA for body but fails AAA, fails AA for small text. | Compute contrast ratio. Flag if body text on bone background uses `#5A5548` or lighter for text below 18px. | Use `#191817` or `#2A2924` for all body text on cream. Reserve `#5A5548` for muted secondary only. |
| "Bone + dark brown" dark mode | Dark mode using `#2A2520` text on `#1A1714` background — warm browns that fail contrast. The bone palette leaking into dark mode with worse contrast. | Dark mode (background luminance below 0.2) with text color in the warm-brown range (`#1A1000`–`#3D2E20`) or beige text (`#E8D8C0`–`#F0E8D8`). | Use a tinted near-black background (`oklch(15% 0.02 250)`) and pure off-white text (`#FAFAFA`). Don't ship warm-brown-on-warm-brown dark mode. |
| Gradient text | Decorative. Reduces scannability. Common on headings and metrics. | Check for `background-clip: text` + gradient background. | Use solid colors for all text. |
| Neon box-shadow glows on dark backgrounds | Cyberpunk-by-default. | Check for `box-shadow` with spread > 10px and high saturation color on dark elements. | Use subtle, purposeful lighting. Remove glows. |
| shadcn/ui default gray palette | Every AI-generated site ships with the same gray scale. | Check for exact shadcn gray values (e.g., `hsl(0 0% 3.9%)` through `hsl(0 0% 98%)`). | Customize color tokens. Tint grays toward brand hue. |

### Contrast Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Dark mode with low-contrast body text | Generated dark themes routinely fail WCAG AA. | Compute contrast ratio between text and background. Flag if < 4.5:1 for body text. | Ensure 4.5:1 contrast. Tint gray toward brand hue. |
| Gray text on colored backgrounds | Washed out, hard to read. | Check text color on colored backgrounds. Compute contrast. Flag if < 4.5:1. | Use darker shade of background color, or white/near-white. |
| Pure black (#000000) background | Harsh, unnatural. | Check for exact `#000000` or `rgb(0,0,0)` on backgrounds. | Tint toward brand hue (e.g., oklch(12% 0.01 250)). |
| Low contrast text (general) | Text fails WCAG AA contrast requirements. | Compute luminance ratio. Flag body text < 4.5:1, large text < 3:1. | Increase contrast between text and background. |

### Color Non-Choices

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Defaulting to dark mode | Retreat from a decision. Safe "modern" default. | Check if background is dark without explicit user/system preference handling. | Choose light or dark based on context and brand. Support user preference. |
| Defaulting to light mode for safety | Same problem inverted. Also a retreat. | Check if light theme is the only option. | Pick deliberately. Support user preference. |

## Layout

### Template Patterns

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Everything centered | Symmetrical, safe, no opinion. | Check `text-align` across major elements. Flag if >80% are `center`. | Left-align body text. Use asymmetric layouts. Center only hero and CTAs. |
| Badge above hero H1 | AI template default. | Check for small pill/badge element immediately preceding the H1 in DOM order. | Remove badge. Or make it serve a genuine purpose with distinctive styling. |
| Copy-paste section templates | Hero → metrics → features → testimonials → pricing → CTA. | Count distinct section layout patterns. Flag if <3 patterns across 6+ sections. | Vary section structures. Mix narrative with feature grids. Break the conveyor belt. |
| Hero metric layout | Big number + small label × 3–4. | Check for elements with very large font-size adjacent to very small text, repeated in a row. | Only show real, meaningful, verifiable metrics. Give them room or remove them. |
| Monotonous spacing | Same gap everywhere. No rhythm. | Compute standard deviation of spacing values. Flag if <2 unique values across sections. | Tight grouping for related items (8–12px). Generous gaps between sections (48–96px). |
| Mac mockup or isometric 3D illustration as hero image | Floating laptop/screenshot with slight 3D tilt + soft shadow. Default v0 / Lovable / Bolt hero visual. | Hero section contains an `<img>` with class matching `/mockup|screenshot|hero-image|product-shot/` AND `transform: rotate(...)` or `perspective` applied. | Use a real product screenshot, no rotation. Or omit the hero image entirely and let the headline carry. |
| Marquee logo carousel | Infinite-scrolling row of grayscale client logos. Animated, distracting, never informative. | CSS `@keyframes` animating `transform: translateX(...)` on a row of `<img>` elements with `filter: grayscale(...)`. | Static logo grid. Or remove entirely if logos don't link to case studies. |
| Animated count-up stats | "10M+ users" with JS counter animation from 0. Distracting, slows page, no informational value. | JS or CSS animation that animates a numeric value from 0 to a target over time. | Display the final number. Skip the animation. |
| Generic abstract geometric / aurora blobs as hero illustration | Blurred colored blobs as hero illustration. Same on every site. | Hero `<div>` or `<svg>` with `filter: blur(...)` and high-saturation fill colors, no real content. | Real product screenshot. Real photography. Or strong typography alone. |

### Card Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Colored left/top border on cards, blockquotes, or code blocks | Most reliable AI design tell. Also leaks onto blockquotes (`blockquote { border-left: 4px solid #10B981; }`) and code blocks (`pre { border-left: 3px solid var(--accent); }`). The colored vertical stripe is the universal LLM "I tried to make it look designed" reflex. | Check for `border-left` or `border-top` with colored value on elements with `border-radius` (cards) OR on `blockquote` / `pre` / `code` elements. Flag if thick (>2px) and colored. | Remove border. Use subtler accents (background tint, italic, quote mark) or none. For code blocks: use background color alone. For blockquotes: use indentation + italic, no border. |
| Identical feature cards (icon + heading + text) × N | Default AI homepage. | Count sibling elements with identical dimensions, same child structure, same padding. Flag if 3+ identical siblings. | Vary card sizes and content density. Mix full-width and half-width. |
| Cards inside cards (Cardocalypse) | Visual noise from nesting. | Check DOM depth of elements with border/shadow/background. Flag if cards nested 3+ levels deep. | Flatten hierarchy. Use spacing, typography, dividers. |
| Wrapping everything in cards | Every content block gets a bordered container. | Count bordered elements vs total content blocks. Flag if >70% are card-wrapped. | Use spacing and alignment for grouping. Cards only for interactive/elevated content. |
| Rounded rectangles with generic drop shadows | Safest, most forgettable shape. | Check for elements with both `border-radius` > 8px and identical `box-shadow` values. Flag if many share the same shadow. | Vary shadow depth by elevation level. Use borders, backgrounds, or nothing. |
| Massive icon containers | Decoration bigger than message. | Compare icon container dimensions to adjacent text content. Flag if icon area > text area. | Side-by-side icon + heading. Or icon in flow without container. |
| Icon tile stacked above heading | Universal AI feature-card template. | Check for rounded-square container directly above a heading element in DOM. | Vary the pattern. Side-by-side layout, or no container. |
| Border accent on rounded element | Thick accent border clashes with rounded corners. | Check for `border` > 2px colored on elements with `border-radius` > 8px. | Remove border or remove radius. They clash visually. |

### Sequence Patterns

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Numbered "1, 2, 3" step sequences | Conveyor-belt layout. | Check for sibling elements containing sequential numbers (1, 2, 3) as content or aria-labels. | Use timeline, narrative, comparison, accordion — anything but numbered boxes. |
| Stat banner rows | Row of unverifiable metrics. | Check for sibling elements with large number + small label pattern in a horizontal row. | Only include real, meaningful metrics. Give them proper space or remove. |
| Sidebar/nav with emoji icons | Emoji as icon substitute. No design system. | Check for emoji characters in navigation or sidebar text content. | Use real icon library (Phosphor, Tabler, Lucide) or skip icons. |
| All-caps section headings | SHOUTING. Hard to read. | Check for `text-transform: uppercase` on heading elements. | Reserve for short labels (2–3 words). Title/sentence case for headings. |

### Structural Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Line length too long (above ~80ch) | Eye loses place tracking back. | Measure line length on text blocks. Flag if > 80ch average. | Add max-width 65ch–75ch to text containers. |
| Skipped heading levels | Breaks document outline for screen readers. | Check heading hierarchy. Flag if h1 → h3 without h2, etc. | Use proper heading hierarchy. No skipping. |
| Cramped padding | Text too close to container edge. | Check padding on bordered/colored containers. Flag if < 8px. | Minimum 8px padding (12–16px ideal) inside bordered containers. |
| Tiny body text (below 12px) | Hard to read, especially on high-DPI screens. Often paired with low-contrast muted color (`#737373` / `text-muted`) making the problem worse. | Check `font-size` on body text. Flag if < 12px. Flag if < 14px on text colored `#737373` or lighter. | Minimum 14px for body content. 16px ideal. Never ship 11-12px gray meta text — it's the AI's "I forgot to write real copy so here's a caption" reflex. |
| Justified text | Creates rivers of whitespace on screens. | Check for `text-align: justify`. Flag if present. | Use `text-align: left` for body text. |

### Structural Problems (continued)

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Useless sticky header with in-page anchor buttons only | Sticky top nav with 4-5 buttons that all scroll to sections on the same page (`#detector`, `#tells`, `#models`). No real navigation. No way to leave the page. Pure scroll-jumping theater. | Sticky/fixed header (`position: sticky` or `fixed`) containing only anchor links (`href^="#"`) and no real outbound links. | Cut the sticky header. Or replace with real navigation (link to docs, blog, signup, source). If the page genuinely needs in-page anchors, put them in a sidebar or table of contents, not a fake nav bar. |
| Footer meta-commentary (version strings, "this demo deliberately avoids...") | Footer text that comments on the design itself instead of carrying real entity info. Includes "This demo deliberately avoids the bone palette. Background #FFFFFF..." style captions, version strings like "slop-buster v1.7.0 · MIT · 2026", and "Built with X" attributions. The footer should carry real entity name, real license, real contact info — not narrate its own design choices. | Footer text matching `/deliberately avoids/i` OR `/this demo (deliberately|avoids)/i` OR `/background #/i` OR `/accent #/i` OR version strings matching `/v\d+\.\d+\.\d+.*·.*MIT/i` in footer. | Real entity name. Real city. Real license. Or omit. Cut any footer text that comments on the design itself. |


## Motion & Interaction

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Bounce or elastic easing | Dated and tacky. | Check for `cubic-bezier` values matching bounce/elastic patterns, or `animation-timing-function: bounce`. | Use ease-out-quart `cubic-bezier(0.25, 1, 0.5, 1)` or ease-out-expo `cubic-bezier(0.16, 1, 0.3, 1)`. |
| Over-animation | Motion without meaning. | Count animated elements. Flag if >50% of visible elements have animations. | One orchestrated entrance. High-impact moments only. Stillness elsewhere. |
| Glassmorphism everywhere | LLM default since 2022. Decorative, not functional. | Check for `backdrop-filter: blur()` combined with semi-transparent backgrounds on multiple elements. | Use only where it solves a real layering problem. Otherwise, solid backgrounds. |
| Every button is primary | No hierarchy. Everything shouts. | Count button elements. Check if >50% share the same visual weight (same bg color, same size). | One primary action per view. Use ghost, text-link, secondary styles for hierarchy. |
| Animating width/height/padding/margin | Layout thrash and janky performance. | Check CSS transitions/animations for layout properties. Flag if width, height, padding, margin are animated. | Use transform and opacity. Or `grid-template-rows` for height. |

## Content & Copy

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Vague aspirational headline | Could describe any company. Communicates nothing. | LLM-based check: can this headline appear on 100+ different websites? If yes, it's slop. | State what the product does, specifically. Name the problem solved. |
| Generic verb-first hero copy ("Build / Ship / Scale / Transform / Unlock") | The LLM's reflex when given no brief. Could describe any company. | Hero H1 starts with one of: Build, Ship, Scale, Transform, Unlock, Supercharge, Empower, Streamline, Harness, Leverage, Reimagine, Elevate, Accelerate, Amplify. | State what the product does, specifically. E.g. "Your deploy pipeline runs in 12 seconds." |
| Aphoristic-cadence headline ("Not X. *Y.*") | Two-clause headline landing on a short italic-serif rebuttal. Anthropic's `impeccable.style` flags this as "Aphoristic-cadence copy." | Regex on hero/subhead text for pattern `^[A-Z][^.]{3,30}\.\s+<[em]i?>[A-Z][^.]{1,30}\.</[em]i?>` | Say what the product does. Don't perform wit. |
| Footer "Made with ♥ in [city]" or "© 2026 [Generic]" | Default LLM footer. Universal across generated sites. | Footer text matching `/Made with [♥❤] in [A-Z][a-z]+/i` OR `/© \d{4} [A-Z][a-z]+/i` with no real entity name. | Real entity name. Real city. Real license. Or omit. |
| Redundant UX writing | Label + sublabel + helper text all saying the same thing. | Check sibling elements inside a container. If text content overlap >70%, flag. | Say it once, say it well. Pick the clearest phrasing. |
| Modal abuse | Complex content in a modal. | Check modal elements. Flag if scrollable, multi-column, or containing >5 interactive elements. | Complex content deserves its own page. |
| AI-generated stock imagery | Too-perfect, plastic quality. Abstract 3D blobs. | LLM-based check on images. Look for telltale smoothness and symmetry. | Real photography, purposeful illustration, or strong typography alone. |
| Fake/generic testimonials | Vague names, no company, no photo. | Check for testimonials without company names, without real photos, with formulaic quotes. | Real names, real companies, real photos. Or remove until you have them. |
| Amputated features on mobile | Critical functionality hidden on mobile. | Check for `display: none` or `visibility: hidden` on key interactive elements at mobile breakpoints. | Adapt the interface to context. Don't strip features. |

## Post-Q1 2026 Frontier-Model Tells

These are the newer patterns that replaced the obvious purple-gradient tell. They look more tasteful, but they still expose the model's default distribution: editorial affect, random emphasis, and premium abstraction without product detail. Includes the bone palette and italic word-accents named verbatim in Anthropic's Opus 4.8 prompt guide.

### Editorial Typography Tells

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Random italic hero word | One serif italic word appears inside an otherwise sans-serif hero headline with no semantic reason. It signals "premium editorial" without meaning. The Opus 4.8 prompt guide names this verbatim as "italic word-accents." | In `h1` or `h2`, find an `em`, `i`, or italic-styled span containing one to three words while the rest of the heading is upright. Flag if the italic word is an abstract noun (trust, clarity, alignment, momentum, craft, signal, intelligence, velocity, resilience, future, commerce, workflows). | Remove the italic treatment unless the word changes tone or meaning. If emphasis is needed, earn it with copy or layout, not typographic seasoning. |
| Decorative bold or italic body emphasis | Body copy bolds or italicizes phrases that do not need emphasis. The formatting does the writing's job. Sister pattern to italic hero word. | Find `strong`, `b`, `em`, or `i` inside paragraphs where the emphasized span is an abstract phrase ("actually moves the needle", "long-term growth", "real outcomes"). | Rewrite the sentence so emphasis is unnecessary. Use formatting only for terms, titles, warnings, or genuinely contrastive emphasis. |
| Unicode faux formatting | Mathematical Unicode letters render as 𝗯𝗼𝗹𝗱 or 𝘪𝘵𝘢𝘭𝘪𝘤 instead of semantic HTML. | Detect characters in `U+1D400–U+1D7FF`, plus decorative arrows `→`, multiplication separators `×`, or repeated dot separators `·` in prose. | Use semantic HTML for real emphasis. Remove decorative Unicode in product copy, docs, and professional prose. |
| Fraunces / Instrument Serif / Playfair as hero display | The 2026 italic-serif hero reflex. Anthropic's Opus 4.7/4.8 house style. Impeccable: "Anthropic/Google's 28-34% italic-serif-display slop on editorial/food briefs." | Hero `font-family` matches `Fraunces`, `Instrument Serif`, `Playfair Display`, or `Crimson Pro`. | If you want serif display, pick a less-loaded face (Tiempos, GT Sectra, Canela, Newsreader). Or commit to a sans-only hero. |
| Tracking-tight oversized hero headline | `letter-spacing: -0.04em` or tighter on a 64-96px hero. The "modern premium" reflex. | Computed `letter-spacing` below -0.03em on elements with `font-size` above 56px. | Use -0.01 to -0.02em on large headings only. Or zero tracking and let the typeface breathe. |
| Section eyebrows as italic-serif kicker labels | "✦ *Introducing* / [Product name]" — italic-serif tracked-letterspaced labels above section headings. | Small text element with `font-style: italic`, `text-transform: uppercase` or `letter-spacing` above 0.08em, in a serif font, immediately preceding a heading. | Cut the eyebrow. Or replace with a numbered section marker (01 / 02 / 03) only if the section is genuinely sequential. |

### Semantic Hollowing Tells

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Premium abstract word soup | `clarity`, `trust`, `momentum`, `alignment`, `signal`, `intelligence`, `infrastructure`, `orchestration`, `velocity`, `resilience` cluster without specifics. | Flag headlines or sections containing 4+ abstract prestige nouns and no concrete product nouns, numbers, named tools, or user actions. | Replace abstractions with operational detail. Say who does what, with which input, and what changes. |
| Specific-looking fake metrics | Precise numbers appear with no source, timeframe, sample, or mechanism. | Detect percentages, multipliers, or large counts near claims with no nearby source/link/date. | Add source, timeframe, and measurement method. If you cannot verify the number, remove it. |
| Surface polish with no scar tissue | The artifact looks refined but contains no lived constraints, exceptions, failed attempts, or concrete mechanism. | Manual check: could the same copy appear on 50 unrelated SaaS sites? Are there no named users, workflows, tools, failures, or limits? | Add scar tissue: real constraints, named workflows, before/after specifics, failure modes, and tradeoffs. |

### Bone Palette Tells (Opus 4.7/4.8 House Style)

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Bone / cream / off-white background as default | The post-2025 Opus 4.7/4.8 house style. The 2026 replacement for VibeCode Purple. Anthropic's prompt guide names it verbatim: "warm cream/off-white backgrounds (~#F4F1EA)." Identical across vet clinic, fintech, and crypto. | Background hex within ±3 points of `#F4F1EA`, `#F4F3EE`, `#F6F5F2`, or `#FAF8F2` AND body text in `#191817`–`#37352F` warm-charcoal range AND accent in terracotta (`#C96442` / `#C8643C`), sage (`#8A9A7B`), dust rose (`#C9A8A0`), or amber (`#C98A42`). | Commit to a real brand color. | Warm-charcoal text on cream failing WCAG AA | The bone palette's accessibility failure mode. `#5A5548` "warm gray" text on `#F4F1EA` cream is contrast 5.0:1 — passes AA for body but fails AAA, fails AA for small text below 18px. | Compute contrast ratio.
