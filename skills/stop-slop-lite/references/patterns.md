# Design Slop Patterns (Lite)

The 35 most important AI-generated design patterns with detection criteria. Curated from the full ~80-pattern catalog in stop-slop-professional. These patterns are not individually bad — they become slop when used by default, without deliberate choice, creating the "generated by a chat interface without an opinion" look.

## Typography

### Overused Fonts

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Inter used for everything | The Helvetica of the LLM era. Every generated page defaults to it. | Check `font-family` computed style. Match against `Inter`, `Inter var`. | Pick a distinctive typeface. Geist, Untitled Sans, Söhne, Inktrap, Haas Grotesk for sans-serif. Tiempos, GT Sectra, Freight Text for serif. |
| Fraunces / Instrument Serif / Playfair as hero display | The 2026 italic-serif hero reflex. Anthropic's Opus 4.7/4.8 house style. Impeccable: "Anthropic/Google's 28-34% italic-serif-display slop on editorial/food briefs." | Hero `font-family` matches `Fraunces`, `Instrument Serif`, `Playfair Display`, or `Crimson Pro`. | If you want serif display, pick a less-loaded face (Tiempos, GT Sectra, Canela, Newsreader). Or commit to a sans-only hero. |
| Italic word-accents in headings (random italic hero word) | One serif italic word appears inside an otherwise sans-serif hero headline with no semantic reason. The Opus 4.8 prompt guide names this verbatim: "italic word-accents." Verbal fry in typographic form. | In `h1` or `h2`, find an `em`, `i`, or italic-styled span containing one to three words while the rest of the heading is upright. Flag if the italic word is an abstract noun (trust, clarity, alignment, momentum, etc.). | Remove the italic treatment unless the word changes tone or meaning. If emphasis is needed, earn it with copy or layout, not typographic seasoning. |
| Unicode faux formatting | Mathematical Unicode letters render as 𝗯𝗼𝗹𝗱 or 𝘪𝘵𝘢𝘭𝘪𝘤 instead of semantic HTML. Also catches decorative arrows `→`, multiplication separators `×`, and repeated dot separators `·` in prose. | Detect characters in `U+1D400–U+1D7FF`, plus decorative arrows `→`, multiplication separators `×`, or repeated dot separators `·` in prose. | Use semantic HTML for real emphasis. Remove decorative Unicode in product copy, docs, and professional prose. |
| Tracking-tight oversized hero headline | `letter-spacing: -0.04em` or tighter on a 64-96px hero. The "modern premium" reflex. | Computed `letter-spacing` below -0.03em on elements with `font-size` above 56px. | Use -0.01 to -0.02em on large headings only. Or zero tracking and let the typeface breathe. |
| Single font family for entire page | No typographic hierarchy. Headings, body, labels, buttons all look the same. | Count unique `font-family` values across all elements. If 1, flag. | Pair a distinctive display font with a refined body font. Minimum two families. |
| Font sizes too close together | Flat hierarchy. No visual priority. | Calculate ratio between consecutive font sizes. If max ratio < 1.2, flag. | Use at least 1.25× ratio between size steps. Fewer sizes with more contrast. |

## Color

### AI Default Palette

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Lavender-purple accents | The most recognizable AI color tell of 2023–2025. | Check for colors in the HSL range H: 250–290, S: 50–100%, L: 50–80%. Flag purple-ish accents. | Choose earth tones, cream-and-pink, high-contrast black-and-bright, warm reds, cool greens. |
| Blue-to-purple gradients | Default "modern" gradient. | Check for `linear-gradient` or `radial-gradient` transitioning from blue range to purple range. | Use solid colors or intentional gradients tied to brand palette. |
| Bone / cream / off-white background as default | The post-2025 Opus 4.7/4.8 house style. The 2026 replacement for VibeCode Purple. Anthropic's prompt guide names it verbatim: "warm cream/off-white backgrounds (~#F4F1EA)." Identical across vet clinic, fintech, and crypto. | Background hex within ±3 points of `#F4F1EA`, `#F4F3EE`, `#F6F5F2`, or `#FAF8F2` AND body text in `#191817`–`#37352F` warm-charcoal range AND accent in terracotta (`#C96442` / `#C8643C`), sage (`#8A9A7B`), dust rose (`#C9A8A0`), or amber (`#C98A42`). | Commit to a real brand color. If you want warm cream, pick a different accent (indigo `#5266EB`, orange `#FF6719`) so the cream doesn't read as default Opus. Or name the aesthetic family explicitly. |
| Warm-charcoal text on cream failing WCAG AA | The bone palette's accessibility failure mode. `#5A5548` "warm gray" text on `#F4F1EA` cream is contrast 5.0:1 — passes AA for body but fails AAA, fails AA for small text. | Compute contrast ratio. Flag if body text on bone background uses `#5A5548` or lighter for text below 18px. | Use `#191817` or `#2A2924` for all body text on cream. Reserve `#5A5548` for muted secondary only. |
| "Bone + dark brown" dark mode | Dark mode using `#2A2520` text on `#1A1714` background — warm browns that fail contrast. The bone palette leaking into dark mode with worse contrast. | Dark mode (background luminance below 0.2) with text color in the warm-brown range (`#1A1000`–`#3D2E20`) or beige text (`#E8D8C0`–`#F0E8D8`). | Use a tinted near-black background (`oklch(15% 0.02 250)`) and pure off-white text (`#FAFAFA`). Don't ship warm-brown-on-warm-brown dark mode. |
| Permanent dark mode with medium-grey body text | Background `#0B0B0F`, body text `#9CA3AF`, all-caps section labels in `#A0A0A0`. Krebs audit: 34% of Show HN pages — the single most common tell. | Background luminance below 0.2 AND body text color in `#808080`–`#A0A0A0` range (medium grey). | Use `#D4D4D8` or lighter for body text on dark backgrounds (contrast 9:1+). Or commit to a light theme. |
| Gradient text | Decorative. Reduces scannability. Common on headings and metrics. | Check for `background-clip: text` + gradient background. | Use solid colors for all text. |

## Layout

### Template Patterns

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Everything centered | Symmetrical, safe, no opinion. | Check `text-align` across major elements. Flag if >80% are `center`. | Left-align body text. Use asymmetric layouts. Center only hero and CTAs. |
| Badge above hero H1 | AI template default. | Check for small pill/badge element immediately preceding the H1 in DOM order. | Remove badge. Or make it serve a genuine purpose with distinctive styling. |
| Centered hero with two CTAs (primary + "Talk to sales") | Primary CTA "Start free trial" + secondary ghost button "Talk to sales". Default AI hero button pair. Universal across v0/Lovable/Bolt output. | Centered hero section with 2+ buttons where one matches `/^(Get Started|Try Free|Start Building|Start Free|Sign Up)$/i` and another matches `/^(Talk to Sales|Book a Demo|Contact Sales|Learn More)$/i`. | One CTA. Or two CTAs with specific verbs tied to the product ("Run your first deploy" / "Read the docs"). |
| All-caps eyebrow with dots or slashes | `NEW · AI OPS · PRIVATE BETA` or `SYSTEM // DESIGN // OPS` appears above the hero. It looks designed but usually carries no user value. | Small all-caps text before `h1` containing `·`, `•`, `//`, or 3+ short labels. | Keep only if it is a real category, status, or metadata label. Otherwise cut it. |
| Bento grid of vague benefits | Four to six polished cards describe `Adaptive Intelligence`, `Seamless Integration`, `Actionable Insights`, or similar empty benefits. | Section has 4+ card-like siblings with short headings and descriptions containing vague benefit terms. | Replace benefits with concrete jobs, screenshots, examples, limits, or proof. Vary card structure only when content demands it. |
| Mac mockup or isometric 3D illustration as hero image | Floating laptop/screenshot with slight 3D tilt + soft shadow. Default v0 / Lovable / Bolt hero visual. | Hero section contains an `<img>` with class matching `/mockup|screenshot|hero-image|product-shot/` AND `transform: rotate(...)` or `perspective` applied. | Use a real product screenshot, no rotation. Or omit the hero image entirely and let the headline carry. |
| Animated count-up stats | "10M+ users" with JS counter animation from 0. Distracting, slows page, no informational value. | JS or CSS animation that animates a numeric value from 0 to a target over time. | Display the final number. Skip the animation. |
| Useless sticky header with in-page anchor buttons only | Sticky top nav with 4-5 buttons that all scroll to sections on the same page (`#detector`, `#tells`, `#models`). No real navigation. No way to leave the page. Pure scroll-jumping theater. | Sticky/fixed header (`position: sticky` or `fixed`) containing only anchor links (`href^="#"`) and no real outbound links. | Cut the sticky header. Or replace with real navigation (link to docs, blog, signup, source). If the page genuinely needs in-page anchors, put them in a sidebar or table of contents, not a fake nav bar. |

### Card Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Colored left/top border on cards, blockquotes, or code blocks | Most reliable AI design tell. Also leaks onto blockquotes (`blockquote { border-left: 4px solid #10B981; }`) and code blocks (`pre { border-left: 3px solid var(--accent); }`). The colored vertical stripe is the universal LLM "I tried to make it look designed" reflex. | Check for `border-left` or `border-top` with colored value on elements with `border-radius` (cards) OR on `blockquote` / `pre` / `code` elements. Flag if thick (>2px) and colored. | Remove border. Use subtler accents (background tint, italic, quote mark) or none. For code blocks: use background color alone. For blockquotes: use indentation + italic, no border. |
| Identical feature cards (icon + heading + text) × N | Default AI homepage. | Count sibling elements with identical dimensions, same child structure, same padding. Flag if 3+ identical siblings. | Vary card sizes and content density. Mix full-width and half-width. |

### Sequence Patterns

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Stat banner rows | Row of unverifiable metrics. | Check for sibling elements with large number + small label pattern in a horizontal row. | Only include real, meaningful metrics. Give them proper space or remove. |
| 3-tier pricing with "Most Popular" badge | Default AI pricing page. The badge carries no information. | Check for three pricing cards with one marked "Most Popular" or "Recommended." | Cut the badge unless it encodes real data. Differentiate tiers by real features. |
| FAQ accordion before footer | Default AI page ending. Nobody reads it. | Check for an accordion component immediately before the footer containing question elements. | Move FAQs to a real help page. Or answer the top question inline in the body. |

### Structural Problems

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Tiny body text (below 12px) | Hard to read, especially on high-DPI screens. Often paired with low-contrast muted color (`#737373` / `text-muted`) making the problem worse. | Check `font-size` on body text. Flag if < 12px. Flag if < 14px on text colored `#737373` or lighter. | Minimum 14px for body content. 16px ideal. Never ship 11-12px gray meta text — it's the AI's "I forgot to write real copy so here's a caption" reflex. |
| Useless sticky header with in-page anchor buttons only | Sticky top nav with 4-5 buttons that all scroll to sections on the same page (`#detector`, `#tells`, `#models`). No real navigation. No way to leave the page. Pure scroll-jumping theater. | Sticky/fixed header (`position: sticky` or `fixed`) containing only anchor links (`href^="#"`) and no real outbound links. | Cut the sticky header. Or replace with real navigation (link to docs, blog, signup, source). If the page genuinely needs in-page anchors, put them in a sidebar or table of contents, not a fake nav bar. |

## Motion & Interaction

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Bounce or elastic easing | Dated and tacky. | Check for `cubic-bezier` values matching bounce/elastic patterns, or `animation-timing-function: bounce`. | Use ease-out-quart `cubic-bezier(0.25, 1, 0.5, 1)` or ease-out-expo `cubic-bezier(0.16, 1, 0.3, 1)`. |
| Glassmorphism everywhere | LLM default since 2022. Decorative, not functional. | Check for `backdrop-filter: blur()` combined with semi-transparent backgrounds on multiple elements. | Use only where it solves a real layering problem. Otherwise, solid backgrounds. |

## Content & Copy

| Pattern | Problem | Detection Hint | Fix |
|---------|---------|----------------|-----|
| Vague aspirational headline | Could describe any company. Communicates nothing. | LLM-based check: can this headline appear on 100+ different websites? If yes, it's slop. | State what the product does, specifically. Name the problem solved. |
| Generic verb-first hero copy ("Build / Ship / Scale / Transform / Unlock") | The LLM's reflex when given no brief. Could describe any company. | Hero H1 starts with one of: Build, Ship, Scale, Transform, Unlock, Supercharge, Empower, Streamline, Harness, Leverage, Reimagine, Elevate, Accelerate, Amplify. | State what the product does, specifically. E.g. "Your deploy pipeline runs in 12 seconds." |
| Premium abstract word soup | `clarity`, `trust`, `momentum`, `alignment`, `signal`, `intelligence`, `infrastructure`, `orchestration`, `velocity`, `resilience` cluster without specifics. The model defaults to prestige-copy abstractions when it has no concrete product detail. | Flag headlines or sections containing 4+ abstract prestige nouns and no concrete product nouns, numbers, named tools, or user actions. | Replace abstractions with operational detail. Say who does what, with which input, and what changes. |
| Aphoristic-cadence headline ("Not X. *Y.*") | Two-clause headline landing on a short italic-serif rebuttal. Anthropic's `impeccable.style` flags this as "Aphoristic-cadence copy." Examples: "It's not a feature. *A platform.*" "Days fade. *Dots don't.*" | Regex on hero/subhead text for pattern `^[A-Z][^.]{3,30}\.\s+<[em]i?>[A-Z][^.]{1,30}\.</[em]i?>` | Say what the product does. Don't perform wit. |
| Surface polish with no scar tissue | The artifact looks refined but contains no lived constraints, exceptions, failed attempts, or concrete mechanism. Modern slop is polished but unspecific. | Manual check: could the same copy appear on 50 unrelated SaaS sites? Are there no named users, workflows, tools, failures, or limits? | Add scar tissue: real constraints, named workflows, before/after specifics, failure modes, and tradeoffs. |
| Footer "Made with ♥ in [city]" or version-string meta-commentary | Default LLM footer. Universal across generated sites. Includes "This demo deliberately avoids the bone palette. Background #FFFFFF..." style meta-commentary about the design itself. The footer should carry real entity info, not narrate its own design choices. | Footer text matching `/Made with [♥❤] in [A-Z][a-z]+/i` OR `/© \d{4} [A-Z][a-z]+/i` with no real entity name OR any footer text containing "deliberately avoids" / "this demo" / "this site" / "background #" / "accent #". | Real entity name. Real city. Real license. Or omit. Cut any footer text that comments on the design itself. |
| "Get Started" / "Try Free" / "Start Building" CTAs with arrow icon | The model's default CTA vocabulary. Could be any product. | Button text matching `/^(Get Started|Try Free|Start Building|Start Free|Try .* Free|Get .* Free)$/i` AND containing an arrow icon (`→`, `→`, `➜`, `↗`). | Use a verb specific to the product. "Run your first deploy" beats "Get Started." |
| "Trusted by 10,000+ companies" with no logos | Unverifiable claim. No logos that link. | Check for trust claims with no corresponding linked logo strip. | Cut the claim. Or link to a real customer list with case studies. |
